WHAT THIS PROJECT DOES
  A collision risk prediction system for vehicles. It reads distance and speed
  data from sensors, calculates how many seconds until collision (TTC), and
  displays live alerts on a dashboard. Works with real hardware or simulation.

QUICK START
  1. Create virtual environment: python -m venv ttc_env
  2. Activate it: ttc_env\Scripts\activate (Windows)
  3. Install: pip install -r requirements.txt
  4. Run dashboard: run_dashboard.bat

IMPORTANT FILES (You'll use these)
  README.md                 - Full project guide (start here)
  run_dashboard.bat         - Launches the dashboard
  src/config.py             - All settings and thresholds
  src/dashboard.py          - Main application
  firmware/main.ino         - Arduino program
  docs/serial_protocol.md   - Data format specification
  validation/protocol_contract_test.py - Test everything works

OTHER USEFUL FOLDERS
  src/                      - Python application code

  bridge/                   - Simulator connection
  LOGS/                     - Recorded sessions
  MODELS/                   - Optional ML model
  validation/               - Validation & testing scripts
  tests/                    - Unit tests

HOW TO GET HELP
  1. Read README.md first

COMMON TASKS

  Test with fake data: python src/serial_simulator.py
  Run all tests:       python validation/protocol_contract_test.py
  Edit settings:       Open src/config.py

DATA FORMAT (Never change this!)
  7 fields: timestamp_ms, distance_cm, speed_kmh, ttc_basic, ttc_ext, 
            risk_class, confidence

RISK LEVELS
  SAFE (0)      = TTC > 3.0 seconds
  WARNING (1)   = 1.5 to 3.0 seconds
  CRITICAL (2)  = TTC <= 1.5 seconds

KEY SETTINGS
  Location: src/config.py
  RISK_THRESHOLDS    - Danger levels
  SERIAL_CONFIG      - Port and speed
  DASHBOARD_CONFIG   - UI settings
